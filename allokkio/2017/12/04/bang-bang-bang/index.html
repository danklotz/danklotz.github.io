Basics

rlang has multiple constructor-functions that can be used instead of the base Rcombine operator c(). These are: lgl(...), int(...), dbl(...), cpl(...), chr(..., .encoding = NULL), bytes(...) and ll(...). The idea here is that this way the code is more explicit than using c() (The resulting data can be inferred from the functions). Most of these funcitons are IMO self-explanatory. The ones that were not for me, are the cpl(...) function which defines a “complex” and the ll(...) function, which defines a “list”, e.g.

print('A logical vector:')
## [1] "A logical vector:"
lgl(1,2,3)
## [1] TRUE TRUE TRUE
print('A complex vector:')
## [1] "A complex vector:"
cpl(1,2,3)
## [1] 1+0i 2+0i 3+0i
print('A list:')
## [1] "A list:"
ll(1,2,3)
## [[1]]
## [1] 1
## 
## [[2]]
## [1] 2
## 
## [[3]]
## [1] 3

rlang comes with a full fleded error handling scheme. The funcitons abort(), warn() and inform “replace” the base functions base::stop(), base::warning() and base::message(). The rlang functions can be more easily adapted to different types.

The functiion as_function() converts an objects to a function. It fetches the function by name if the input is supplied as a string and transforms a quosre to a function, e.g.

print('example from the rlang documentation:')
## [1] "example from the rlang documentation:"
f <- as_function(~ . + 1)
f(10)
## [1] 11
# does not worK:
#  ff <- as_function("~ . + 1")

print('fetch function with the name f:')
## [1] "fetch function with the name f:"
ff <- as_function("f")
ff(10)
## [1] 11
print('construct closue:')
## [1] "construct closue:"
as_closure(~ . + 1)
## function (..., .x = ..1, .y = ..2, . = ..1) 
## . + 1

rlang has an operator to get the infix attributes, i.e. %@%. It wraps around the attr(x,name) function so that it can be used as infix function. The function does not accept quosures, but could still be handy- The examples in the documentation are:

# the function:
print(`%@%`)
## function (x, name) 
## {
##     attr(x, name, exact = TRUE)
## }
## <environment: namespace:rlang>
# the example:
factor(1:3)
## [1] 1 2 3
## Levels: 1 2 3
factor(1:3) %@% "levels"
## [1] "1" "2" "3"
# equivalent: attr(factor(1:3),"levels") 
mtcars %@% "class"
## [1] "data.frame"

The functions sym(x) and syms(x) can be used to create symbols out of strings or lists of strings, e.g.

print("create symbol from string")
## [1] "create symbol from string"
x <- 3
"x" %>% sym() %>% eval_tidy() 
## [1] 3
#
print("create list of symbols from string")
## [1] "create list of symbols from string"
x <- list(a = 1, b = 2, c = 3)
"x" %>% sym() %>% eval_tidy() 
## $a
## [1] 1
## 
## $b
## [1] 2
## 
## $c
## [1] 3

An Example for a promise.

Ok, first an example from dplyr.tidyvesre.org

df <- tibble(
  g1 = c(1, 1, 2, 2, 2),
  g2 = c(1, 2, 1, 2, 1),
  a = sample(5), 
  b = sample(5)
)
quo(g1)
## <quosure: global>
## ~g1
#> <quosure: global>

quo(a + b + c)

#> <quosure: global>

Does work:

my_summarise <- function(df, group_var) {
  df %>%
    group_by(!!group_var) %>%
    summarise(a = mean(a))
}

my_summarise(df, quo(g1))
## # A tibble: 2 x 2
##      g1        a
##   <dbl>    <dbl>
## 1     1 3.500000
## 2     2 2.666667

Does not work:

my_summarise <- function(df, group_var) {
  group_var %<>% quo(group_var)
  df %>%
    group_by(!!group_var) %>%
    summarise(a = mean(a))
}

my_summarise(df, g1)

In order for it to work we need a promise. Within the tidy evaluation framework a promise can be made with the enquo() function.

my_summarise <- function(df, group_by) {
  group_by <- enquo(group_by)
  df %>%
    group_by(!!group_by) %>%
    summarise(a = mean(a))
}

my_summarise(df, g1)
## # A tibble: 2 x 2
##      g1        a
##   <dbl>    <dbl>
## 1     1 3.500000
## 2     2 2.666667
